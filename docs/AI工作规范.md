# 智算系统 - AI 工作规范

## 工作前必做检查

每次打开智算文件夹工作时，必须先执行：

```bash
cd /path/to/智算
git fetch origin           # 拉取服务器最新状态（不合并）
git status                # 查看本地是否有未提交的改动
git log --oneline -3      # 查看最近3次提交
```

如果发现：
- **有未提交的改动**：先处理完再继续
- **有新的远程提交**：告知用户最新改动内容

## 修改记录规则

每次修改代码后，必须记录以下信息：

### 1. Git 提交格式

```
[类型] 简短描述

详细说明：
- 修改了哪些文件
- 为什么要改
- 修改前后的区别（如果有）

示例：
[新增] 添加中际联合股票配置

详细说明：
- 修改文件：config/stocks.py
- 添加中际联合股票代码 sh605305
- 原因：原配置代码错误
```

### 2. 类型标识

| 类型 | 用途 |
|------|------|
| [新增] | 添加新功能 |
| [修改] | 修改现有功能 |
| [修复] | 修复Bug |
| [优化] | 性能或代码优化 |
| [文档] | 文档更新 |
| [配置] | 配置更改 |

### 3. 必须记录的情况

- 修改任何 .py 文件
- 修改配置文件
- 添加新股票
- 修改趋势规则
- 修改数据获取逻辑

### 4. 简化提交示例

```bash
# 好
git add .
git commit -m "[修复] 修正fetcher_ifind.py的代码格式问题"
git push

# 不好
git add .
git commit -m "fix"
git push
```
