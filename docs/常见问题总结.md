## 问题1：down_break → down_rally 转换时 rally_low 未重置

**发现日期**：2026-03-25

**问题描述**：
当趋势从 `down_break`（破碎）转换为 `down_rally`（回撤）时，`rally_low` 参数没有被重置为当前低点，而是沿用了旧值。

**具体案例**：
- 神火股份 3月9日-3月10日
- 3月6日 rally_low = 33.80（当日低点）
- 3月9日 09:31 触发 down_break
- 3月10日 09:30 触发 down_break → down_rally
- 此时 rally_low 应该更新为 34.85（新低点），但程序仍沿用 33.80

**影响**：
- 导致 rally_low 计算错误
- 影响后续 down_rally → down_secondary 的转换判断

**根本原因**：
`core/trend.py` 中 `down_break` 状态的转换逻辑：
```python
elif trend == "down_break":
    elif break_high is not None and n_high is not None and high is not None and high > break_high and high > n_high * 1.03:
        key_high = high
        new_trend = "up"
    elif break_high is not None and low is not None and low < break_high * 0.94:
        new_trend = "down_rally"
        # rally_low 未更新！
```

**修复方案**：
```python
elif break_high is not None and low is not None and low < break_high * 0.94:
    rally_low = low  # 新增：重置 rally_low 为当前低点
    new_trend = "down_rally"
```

**修复文件**：
- `core/trend.py`

**验证结果**：
- 修复后 3月6日、3月10日趋势判断正确率：30/30 = 100%

## 问题2：全市场趋势分析 key_high=None 导致卡死

**发现日期**：2026-03-25

**问题描述**：
全市场趋势分析脚本 `analyze_all_trends.py` 硬编码 `key_high=None`，导致所有股票从第一条数据开始就卡在 `up` 状态无法转换。

**根本原因**：
```python
state = init_state({
    "trend": "up",
    "key_high": None,  # ← 问题所在
    ...
})
```

当 `key_high=None` 时，up 状态的判断条件 `if key_high is not None` 永远为 False，无法触发任何转换。

**影响**：
- 全市场 3667 只股票的趋势判断全部错误
- 所有股票都显示从 2025-01-02 到数据终点始终为 up 状态

**状态**：待修复
