## 智算系统查阅指南

当被问起智算系统时，按以下优先级查阅：

### 查阅优先级

1. **我的记忆** → `MEMORY.md`（汇总版，快速定位）
2. **系统输出** → `quant_v2/output/趋势判断/`（趋势判断、数据）
3. **系统日志/状态** → `quant_v2/` 目录

### 智算2.0 输出链条（追查问题用）

**完整流程**：数据获取 → 趋势判断 → 行情记录 + 实时动态

| 环节 | 输出目录 | 文件类型 | 问题排查 |
|------|----------|----------|-----------|
| 1. 数据获取 | `quant_v2/data/` | `*_min1.csv` | 检查最新时间是否为14:59 |
| 2. 趋势判断 | `quant_v2/output/趋势判断/` | `*_趋势判断.csv` | 检查是否最新日期 |
| 3. 行情记录 | `quant_v2/output/行情记录/` | `*_行情记录.csv` | 检查是否最新日期 |
| 4. 实时动态 | `quant_v2/output/实时动态/` | `*_实时动态.md` | 检查时间是否最新 |

**排查顺序**：如果某个环节出问题，按顺序往上查：
- 实时动态不对 → 查行情记录 → 查趋势判断 → 查数据获取
- 命令：`ls -la 文件名` 查看文件修改时间

### ⚠️ 修改程序代码规则

**最重要**：修改任何程序代码时，**必须先告诉用户**准备干什么、实现什么效果，取得同意后再修改。

### 📦 更新技能步骤

当用户说"更新技能"时，按以下步骤：

1. **备份** - 把当前的技能文件夹复制到 `~/.openclaw/workspace/技能备份/`

3. **更新脚本** - 如果有对应脚本，也一起更新

当执行涉及 skill 的任务时（如检查智算、发截图等），**先读取对应 SKILL.md**，确保按正确方式执行。
